Naming:

  • theme name: WordPress Theme
  • theme unique name: wordPress.theme
  • skin name: WordPress Skin
  • skin unique name: wordPress.skin

Dynamic content:

  • dynamic contents location: <wp_profile_root>/installedApps/node/WordPressTheme.ear
  • dynamic content EAR display name: WordPressTheme
  • dynamic content WAR context root: /wordPressTheme
  • dynamic content spots module name: dyncs_wordpress
  • head dynamic content spot id: wordpress_head
  • navigation dynamic content spot id: wordpress_nav

Static content:

  • static contents location: dav:fs-type1/themes/wordPress/
  • static content module name: static_wordpress

PAA installation instructions:

  1. Copy output/wordPressTheme.paa to your Portal server. In the example below, the file was copied to the /opt directory.
  2. To install the paa, open a command shell and go to the ConfigEngine directory, for example: /opt/WebSphere/wp_profile/ConfigEngine
  3. Run these two tasks, remembering to provide the appropriate passwords:
    • ./ConfigEngine.sh install-paa -DPAALocation=/opt/wordPressTheme.paa -DWasPassword=WAS_PASSWORD -DPortalAdminPwd=PORTAL_PASSWORD
    • ./ConfigEngine.sh deploy-paa -DappName=wordPressTheme -DWasPassword=WAS_PASSWORD -DPortalAdminPwd=PORTAL_PASSWORD

Next steps:

  1. If the WordPress style.css file contains import statements, add them to the DX theme relative to: dav:fs-type1/themes/wordPress/css/style.css
  2. The script automatically adds any /images/ or /img/ folders into the DX theme automatically. If the WordPress them requires other image folders besides these, move them into: dav:fs-type1/themes/wordPress/css/
  3. The script created a dynamic navigation JSP file based on the WordPress theme input. This file can be edited here: <wp_profile_root>/installedApps/node/WordPressTheme.ear/wordPressTheme.war/themes/html/dynamicSpots/navigation.jsp
  4. If desired, edit the theme's <head> section here: <wp_profile_root>/installedApps/node/WordPressTheme.ear/wordPressTheme.war/themes/html/dynamicSpots/head.jsp
  5. The script found straight-forward pieces of PHP code in the theme and either removed or replaced them with appropriate DX theme code. However, the bulk of the PHP code must be handled manually. Use these references to clean up the PHP in the following files:
    • dav:fs-type1/themes/wordPress/theme.html
    • dav:fs-type1/themes/wordPress/skins/Standard/skin.html
  6. The script added jquery to the theme. If this is not needed, remove it from here: dav:fs-type1/themes/wordPress/profiles/profile_deferred.json
  7. See here for adding extra JS and CSS resources to the DX theme.

PAA uninstall instructions:

  1. Open a command shell and go to the ConfigEngine directory, for example: /opt/WebSphere/wp_profile/ConfigEngine
  2. Run these two tasks, remembering to provide the appropriate passwords:
    • ./ConfigEngine.sh remove-paa -DappName=wordPressTheme -DforceRemove=true -DWasPassword=WAS_PASSWORD -DPortalAdminPwd=PORTAL_PASSWORD
    • ./ConfigEngine.sh uninstall-paa -DappName=wordPressTheme -DforceUninstall=true -DWasPassword=WAS_PASSWORD -DPortalAdminPwd=PORTAL_PASSWORD
